# Крайне рекомендуем развернуть решение

TrendWatcher — Python-пайплайн для мониторинга новостных инфоповодов в авто, автокредитовании, дилерском бизнесе и смежной финансовой повестке. Проект собирает новости из RSS и Telegram, очищает тексты, дедублицирует похожие публикации, отбирает релевантные сигналы и формирует аналитические карточки через LLM. Результат сохраняется в `data/trends.json` и может просматриваться через простой веб-интерфейс.

## Что делает проект в кратце

1. **Парсит источники**: RSS-ленты и Telegram-каналы из `parser/sources.py`.
2. **Очищает тексты**: убирает HTML, эмодзи, служебные Telegram-фрагменты и повторы.
3. **Дедублицирует новости**: объединяет похожие инфоповоды текстовыми правилами и, при необходимости, embeddings.
4. **Отбирает новые инфоповоды**: сверяет новости с уже обработанными через `data/seen_infopovods.json`.
5. **Фильтрует релевантность**: по ключевым словам, через LLM или в гибридном режиме.
6. **Создаёт карточки сигналов**: генерирует title, summary, why_hot, draft_for_team, topic, tone, entities и hotness.
7. **Показывает карточки в браузере**: Flask-приложение читает `data/trends.json`.

## Структура проекта

```text
trendwatcher/
├── config.py                         # настройки, env-переменные и пути
├── pipeline.py                       # основной пайплайн обработки
├── requirements.txt                  # зависимости Python
├── parser/
│   ├── sources.py                    # RSS-ленты и Telegram-каналы
│   ├── parsers.py                    # логика парсинга RSS/TG
│   └── run_parsers.py                # запуск парсера
├── core/
│   ├── clean_data.py                 # очистка текстов
│   ├── deduplicate.py                # дедубликация
│   ├── filter_new_infopovods.py      # фильтр новых инфоповодов
│   ├── step_llm_filter.py            # фильтр релевантности
│   ├── step_make_card.py             # генерация карточек
│   ├── hotness.py                    # расчёт hotness
│   └── llm.py                        # работа с OpenRouter/LLM/embeddings
├── data/
│   ├── parsed_news.csv               # сырые новости после парсинга
│   ├── cleaned_df.csv                # очищенные новости
│   ├── deduplicated_primary.csv      # новости после дедубликации
│   ├── new_infopovods.csv            # новые инфоповоды
│   ├── auto_only.csv                 # релевантные авто/финансовые инфоповоды
│   ├── trends.json                   # итоговые карточки
│   └── seen_infopovods.json          # история обработанных инфоповодов
└── web/
    ├── app.py                        # Flask-приложение
    ├── templates/index.html          # HTML
    └── static/style.css              # стили
```

## Требования

- Python 3.10+
- API-ключ OpenRouter для LLM и embeddings
- Telegram API ID/API Hash — опционально, нужны только для парсинга Telegram

## Установка

```bash
cd trendwatcher
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск пайплайна

Из корня проекта:

```bash
python pipeline.py
```

Пайплайн выполнит этапы:

```text
0/5 — парсинг
1/5 — очистка текста
2/5 — дедубликация
3/5 — поиск новых инфоповодов
4/5 — фильтр релевантности
5/5 — создание карточек
```

После успешного запуска итоговые карточки будут обновлены в:

```text
data/trends.json
```

## Запуск веб-интерфейса

Веб-приложение читает `data/trends.json`, поэтому сначала запустите пайплайн или убедитесь, что файл уже существует.

```bash
gunicorn -w 2 -b 0.0.0.0:5000 web.app:app
```

По умолчанию приложение будет доступно по адресу:

```text
http://localhost:5000
```

Для постоянной работы запускать надо на сервере и запускать пайплайн через cronlab автоматически раз в час, сначала один раз в ручную запустить пайплайн, запустить сайт, карточки будут обновляться при перезагрузке сайта через F5
